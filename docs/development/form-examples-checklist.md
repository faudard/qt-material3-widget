# Stronger Form Examples Checklist

Use this checklist when building example forms that exercise the widget families together.

## Example goals
- [ ] Show realistic grouped form layouts, not isolated widgets only
- [ ] Demonstrate both outlined and filled text fields
- [ ] Demonstrate helper/supporting text and error states
- [ ] Demonstrate validation-trigger patterns
- [ ] Demonstrate date-field shell integration
- [ ] Demonstrate autocomplete popup integration
- [ ] Demonstrate primary/secondary form actions
- [ ] Demonstrate keyboard-first navigation paths

## Form scenarios
- [ ] Basic account/profile form
- [ ] Validation-heavy sign-up or settings form
- [ ] Date + autocomplete composite flow
- [ ] Prefix/suffix usage in realistic inputs

## Quality rules
- [ ] Example pages use public widget APIs only
- [ ] Example pages reflect recommended layout spacing
- [ ] Examples show light and dark themes cleanly
- [ ] Examples are useful for screenshots and regression checks
- [ ] Examples avoid fake behavior that contradicts the intended widget API

## Architecture value
- [ ] Examples exercise multiple widget families together
- [ ] Examples expose API rough edges early
- [ ] Examples are usable as manual QA surfaces
- [ ] Examples can evolve into docs screenshots later
